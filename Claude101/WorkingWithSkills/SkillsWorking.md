## 🧠 Lesson 1 — Your First Conversation with Claude

### 🎯 The 3 Elements of a Great Prompt

| Element | Question to Ask Yourself | Example |
|---|---|---|
| **Setting the Stage** | What's my role & context? | *"I'm a marketing lead at a startup..."* |
| **Defining the Task** | What action do I want? | *"...research the streaming market..."* |
| **Specifying Rules** | What style/format do I need? | *"...structured as a professional report."* |

### 💡 The "Coworker Approach"
Talk to Claude **naturally, concisely, and conversationally** — like messaging a smart coworker, not typing a Google search.

### 🔄 Iterating on Responses
- Ask for more detail → *"Can you expand on point 2?"*
- Give feedback → *"Good, but make it more casual."*
- Redirect → *"Actually, I meant X, not Y."*

## 🎯 Lesson 2 — Getting Better Results

### 🐛 Common Challenges & How to Fix Them

| Problem | Why It Happens | The Fix |
|---|---|---|
| **Too generic** | Not enough context | Add your role, audience & constraints |
| **Too long / too short** | Claude is guessing at length | Be explicit: *"Keep this under 100 words"* |
| **Wrong format** | Claude knows WHAT but not HOW | Show an example or describe the structure |
| **Wrong information** | Claude can occasionally be wrong | Verify facts independently for high-stakes work |
| **Wrong tone** | Claude defaults to professional | Say exactly what you want: *"More conversational"* |

### 🔁 The Iteration Mindset
> *"Your first prompt is the start of a conversation, not a one-shot request."*

✅ Treat first drafts as **starting points**  
✅ Give **specific feedback** → *"Cut the first two paragraphs"* not just *"Make it shorter"*  
✅ Know when to **start fresh** — sometimes a new chat is faster  

### 🏆 The 4D Framework for AI Fluency

| Competency | What It Means |
|---|---|
| 🎯 **Delegation** | Decide what AI does vs. what humans do |
| 📝 **Description** | Communicate clearly with AI |
| 🔍 **Discernment** | Critically evaluate AI outputs |
| 🛡️ **Diligence** | Use AI responsibly & ethically |

## 📁 Lesson 4 — Introduction to Projects

### What is a Project?
> A **Project** is a self-contained workspace with its own memory, chat history, knowledge base, and custom instructions — like a dedicated office for each area of your work.

### 🤔 When Should I Create a Project?

| Signal | Example |
|---|---|
| 📄 **Repeated reference materials** | Brand guidelines, meeting notes, reports |
| 📋 **Consistent requirements** | Always cite sources, always use our template |
| 👥 **Team collaboration needs** | Multiple people need the same shared context |

### 🚀 How to Create Your First Project — 3 Steps

**Step 1: Set Up Your Project**
1. Go to `claude.ai/projects` → click **"+ New Project"**
2. Give it a descriptive name → *"Q4 Marketing Campaign"*
3. Add a brief description
4. Choose visibility: **Private** or **Share with organization**

**Step 2: Write Project Instructions**
```
✅ Context    → "This project is for B2B software marketing content."
✅ Process    → "First draft a structure, then write the full piece."
✅ Tone       → "Professional but conversational. Avoid jargon."
✅ Rules      → "Always end marketing copy with a call-to-action."
```

**Step 3: Build Your Knowledge Base**
- 📘 Brand guidelines & style guides
- 📊 Research reports & meeting notes
- 📝 Templates & examples of great work
- 🔧 Technical docs & specifications

> 💡 **Pro Tip:** Name files descriptively! `Q4-2024-Brand-Guidelines.pdf` helps Claude — `document1.pdf` doesn't.

### 🧠 RAG — How Projects Handle Large Knowledge Bases
```
Normal Mode  →  Claude loads ALL project content into memory
RAG Mode     →  Claude intelligently retrieves ONLY what's relevant
```
- 🔼 Expands capacity by up to **10x** — quality stays the same

### 👥 Collaboration & Permission Levels
*(Claude for Work — Team & Enterprise plans)*

| Permission | What They Can Do |
|---|---|
| 👁️ **Can View** | Read, access knowledge, chat — no editing |
| ✏️ **Can Edit** | Modify instructions, update knowledge, manage members |
| 👑 **Owner** | Full control including visibility settings |

## 🎨 Lesson 5 — Creating with Artifacts

### What is an Artifact?
> An **Artifact** is a standalone, interactive output Claude creates in a **dedicated window** next to your chat — live and ready to use, not buried in conversation.

### 🗂️ Types of Artifacts

| Type | Best For |
|---|---|
| 📄 **Documents** | Reports, notes, blog posts, templates |
| 💻 **Code Snippets** | Any programming language |
| 🌐 **HTML Pages** | Web pages & prototypes |
| 🖼️ **SVG Images** | Logos, icons, illustrations |
| 📊 **Mermaid Diagrams** | Flowcharts, org charts, Gantt charts |
| ⚛️ **React Components** | Interactive UI with real logic |

### 📤 Sharing Options

| Option | Who It's For |
|---|---|
| 📋 **Copy / Download** | Personal use |
| 🏢 **Share within org** | Team & Enterprise users |
| 🌍 **Publish publicly** | Free, Pro & Max — anyone can view & remix |

> 💡 **Specific > Vague:** *"Budget tracker with categories, pie chart & over-budget warning"* beats *"budget tracker"* every time.

## 🛠️ Lesson 6 — Working with Skills

### What is a Skill?
> A **Skill** is a folder of instructions, scripts, and resources that Claude loads automatically to perform specialized tasks in a **repeatable, consistent way** — like an expertise package.

You've already seen Skills in action whenever Claude creates Excel, PowerPoint, Word, or PDF files!

---

### 📦 Two Types of Skills

| Type | Created By | How It Works |
|---|---|---|
| 🔵 **Anthropic Skills** | Anthropic | Built-in, auto-invoked, available to all paid users |
| 🟣 **Custom Skills** | You / Your org | Tailored to your workflows, uploaded by you |

**Anthropic's built-in Skills include:**
- 📊 Excel spreadsheet creation
- 📝 Word document creation
- 📊 PowerPoint presentation creation
- 📄 PDF generation

---

### ⚙️ How to Enable Skills

1. Go to **Settings → Capabilities**
2. Toggle on **"Code execution and file creation"** *(Skills require this)*
3. Scroll to the **Skills section**
4. Toggle individual skills **on or off** as needed

> ⚠️ **Enterprise:** Owners must enable Skills in Admin settings first  
> ✅ **Team plans:** Enabled by default at the organization level

---

### 💬 Example Prompts That Invoke Skills

```
"Create an Excel spreadsheet tracking monthly expenses with totals"
"Turn these meeting notes into a PowerPoint presentation"
"Generate a PDF report summarizing this data"
"Build a financial model in Excel with scenario analysis"
```
When Claude uses a Skill, you'll see it mentioned in its chain of thought — and the output will be a **downloadable file** you can save or send to Google Drive.

---

### 🧰 Creating Your Own Custom Skill — 5 Steps

1. **Start a chat** → Tell Claude what skill you want to create
   - *"I want a skill for writing quarterly business reviews"*
2. **Answer Claude's questions** → It will interview you about your workflow
3. **Upload reference materials** → Templates, style guides, brand assets, examples
4. **Download your skill** → Claude generates a ZIP file with the skill structure
5. **Upload to Settings** → Settings → Capabilities → Skills → "Upload skill"

From that point on, Claude **automatically invokes it** whenever relevant — no manual triggering needed!

---

### 🆚 Skills vs. Projects — What's the Difference?

| | 📁 Projects | 🛠️ Skills |
|---|---|---|
| **Purpose** | Store **knowledge** Claude references | Define **processes** Claude executes |
| **Think of it as...** | A filing cabinet | A recipe book |
| **Best for** | Long-term context, reference materials | Repeatable workflows, consistent methodology |
| **Example** | Customer hub, research library | Brand guidelines applier, blog drafter |
| **Persistence** | Available across all chats in the project | Applied whenever the skill is invoked |

> 💡 **They work together!** A skill can pull knowledge stored in a project — the project provides the **WHAT** (information), the skill provides the **HOW** (process).

---

### 🔒 Security Considerations

- ✅ Only install custom Skills from **trusted sources**
- ✅ Anthropic's built-in Skills are **tested and maintained** by Anthropic
- ✅ Your custom Skills are **private** to your individual account
- ⚠️ Always **review contents** of external Skills before installing
