# Making Changes with Claude

> A practical guide for implementing changes in your development environment using Claude's key features.

---

## 📸 Using Screenshots for Precise Communication

Paste a screenshot directly into the chat with **`Ctrl+V`** to show Claude exactly what you want to change. This is the most effective way to communicate UI or layout modifications.

---

## 🗂️ Planning Mode

Best for tasks requiring **broad codebase understanding** or multi-file changes.

**Activate:** Press `Shift + Tab` twice (or once if auto-accepting edits)

Claude will:
- Read more files across your project
- Create a detailed implementation plan
- Wait for your approval before making any changes

---

## 🧠 Thinking Modes

Best for **complex logic, debugging, or algorithmic challenges**.

| Mode | Depth |
|------|-------|
| `Think` | Basic reasoning |
| `Think more` | Extended reasoning |
| `Think a lot` | Comprehensive reasoning |
| `Think longer` | Extended time reasoning |
| `Ultrathink` | Maximum reasoning capability |

---

## ⚖️ When to Use Each

| Scenario | Use |
|----------|-----|
| Multi-file changes, broad codebase tasks | Planning Mode |
| Complex logic, hard bugs, algorithms | Thinking Mode |
| Both breadth and depth needed | Combine both |

> **Note:** Both modes consume additional tokens — use them when the complexity justifies it.
